# Xross-Stars Solo Practice Tool

## 概要
本アプリは、カードゲーム「Xross-Stars」の一人回し練習を目的としたWebアプリケーションです。  
1つの画面上で2つのデッキを操作できるようにし、対戦の盤面確認や動きの練習をしやすくすることを目的として作成しました。

通常の対戦では相手プレイヤーが必要になりますが、このアプリでは1人で2つのデッキを操作できるため、特定の盤面や手順の練習を効率よく行うことができます。

工夫した点
1. 一人で2デッキを同時に扱えるUIにした点

通常のカードゲーム用ツールでは片側の操作だけを想定したものも多いですが、本アプリでは1人で両方のデッキを操作できるようにし、練習用ツールとして使いやすい構成にしました。

2. ルールをそのまま机上再現するのではなく、ツール向けに簡略化した点

実際のルールではダメージカウンターを使いますが、本アプリでは盤面確認をしやすくするため、HPを直接増減する方式にしました。
これにより、練習中の状況把握がしやすくなっています。

3. タクティクスエリアをモーダル化した点

対戦画面上に常設すると情報量が増えすぎるため、必要な時だけ確認できるモーダル表示にしました。
これにより画面全体の見やすさを保ちつつ、操作もしやすくしました。

4. 装備タクティクスに対応した点

装備タクティクス使用時には、使用後すぐに消費するのではなく、装備先リーダーを選ぶ処理を追加しました。
ゲーム特有のルールを反映したことで、より実践に近い操作ができるようになっています。

5. JSON保存からSQLite保存へ変更した点

当初はJSONファイルで管理していましたが、課題提出や今後の拡張を考えてSQLiteへ移行しました。
これにより、カードやデッキのデータ管理がしやすくなり、DockerやEC2環境でも扱いやすくなりました。

6. Docker + EC2 で動作させた点

ローカルだけでなく、AWS EC2上でDockerコンテナとして起動できる構成にしました。
これにより、環境差異を減らし、他環境でも同じように起動しやすくしています。

苦労した点
1. 状態管理の複雑さ

2つのデッキを同時に操作するため、各プレイヤーの山札、手札、トラッシュ、PP、リーダー状態、タクティクス状態を分けて管理する必要がありました。
特に、ラウンド移行時の処理や勝敗判定の管理が複雑でした。

2. タクティクスの扱い

通常タクティクス、装備タクティクス、PPチケットをそれぞれ適切に扱う必要がありました。
特に装備タクティクスについては、使用時に装備先を選ばせる処理の追加が必要で、通常の消費型タクティクスと分岐して管理しました。

3. 保存方式の変更

途中で保存形式をJSONからSQLiteに変更したため、既存処理をデータベース対応に置き換える必要がありました。
ただしこの変更により、結果として提出向けにより良い構成になったと考えています。

4. Docker / EC2 デプロイ

ローカルでは動いていても、EC2上ではDockerやComposeのバージョン差異、権限、静的ファイルの反映などでつまずく場面がありました。
最終的にはGitHub経由でEC2へデプロイできる構成にできました。

ローカル起動方法
1. リポジトリを取得
git clone https://github.com/charu-hue/xross-stars-tool.git
cd xross-stars-tool
2. Dockerで起動
docker compose up --build
3. ブラウザで確認
http://localhost
EC2デプロイ手順
1. EC2インスタンスを作成
Amazon Linux 2023
セキュリティグループで SSH(22) と HTTP(80) を許可
2. Dockerをインストール
sudo yum update -y
sudo yum install -y docker
sudo service docker start
sudo usermod -aG docker ec2-user
3. GitとDocker Composeを利用可能にする
sudo yum install -y git

Docker ComposeはCLI pluginとして導入した。

4. GitHubから取得
git clone https://github.com/charu-hue/xross-stars-tool.git
cd xross-stars-tool
5. コンテナ起動
docker build -t xross-stars-tool-web .
docker compose up -d
6. Elastic IPでアクセス

ブラウザから Elastic IP にアクセスして動作確認を行う。
